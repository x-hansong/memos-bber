# Selection Quick Save

## What We're Building
在网页中选中文字时，于选区附近浮现一个快捷按钮。用户点击后，插件直接把选中的内容发送到 Memos，并自动附带当前页面链接。

这次需求的关键不只是“抓到文字”，还要保留选区里的链接结构。如果用户选中的内容里包含超链接，保存结果不能退化成纯文本，而要尽量保留原始链接信息，避免现有右键菜单 `selectionText` 那种只剩字面文本的情况。

## Why This Approach
仓库里已经有两条现成路径：一条是右键菜单把内容追加到扩展输入框，另一条是弹窗里的 `sendText()` 直接把文本发到 Memos。当前缺口在于页面内没有“选中即发”的入口，而且 `selectionText` 会丢失 HTML 链接。

考虑过三种方向：
- 延续右键菜单模式，只新增一个更聪明的“发送选区”入口：改动小，但仍然不是用户要的悬浮按钮。
- 页面内显示悬浮按钮，但点击后仍只写入扩展输入框：交互一致，但多了一步确认，不符合“自动保存”。
- 页面内显示悬浮按钮，点击后直接发送到 Memos，同时从 DOM 选区提取富文本里的链接并转换为 Memos 可接受的文本格式：最贴合需求，也是推荐方案。

推荐第三种。它满足目标路径最短，且只在选区场景引入新能力，不需要重做现有 popup 工作流。

## Key Decisions
- 使用页面内悬浮按钮作为入口：比右键菜单更直接，符合“选中文字时浮现按钮”的明确需求。
- 点击后直接发送到 Memos：默认走快速保存，而不是先填充到 popup，再次确认。
- 保存内容中追加当前页面链接：作为选区内容后的固定附加信息，确保 memo 保留来源上下文。
- 选区中的超链接需要保留：不依赖 `selectionText`，而是基于 DOM 选区读取结构化内容，再转成能在 Memos 中保真的文本表示。
- 保持最小范围改动：新功能只覆盖文本选区场景，不顺带扩展图片、复杂富文本编辑器兼容等额外需求。
